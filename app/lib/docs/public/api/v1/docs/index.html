    Introduction

    API documentation for RAPiDSkills. This API follows the JSON:API v1.0 specification.

    Authentication

    After successful sign in, an access token is returned in the response. All authenticated calls are required to pass this access token as a Bearer token in the header of the request. The tokens do not expire, but signing out or changing the user password will invalidate the token.

    Sign in

    Example request

    curl -X POST \
  http://www.rapidskills.org/api/v1/sessions \
  -H 'Accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
    "data": {
        "type": "sessions",
        "attributes": {
            "email": "mickey@example.com",
            "password": "supersecret"
        }
    }
}'

    HTTP Request

    POST http://www.rapidskills.org/api/v1/sessions

    Resource attributes

    Parameter Type Description
    email
    required    		 string
    password
    required    		 string

    Downloads

    PDF/Excel generation

    This endpoint will trigger the PDF and/or Excel document generation for the resource passed in the relationships field.

    Example request

    curl -X POST \
  http://www.rapidskills.org/api/v1/downloads \
  -H 'Accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
    "data": {
        "type": "download",
        "relationships": {
            "downloadable": {
                "data": {
                    "type": "occupation_standard",
                    "id": "1"
                }
            }
        }
    }
}'

    HTTP Request

    POST http://www.rapidskills.org/api/v1/downloads

    Downloadable relationship attributes

    Parameter Type Description
    type
    required    		 string Singular object type for which to generate downloadable items
    id
    required    		 string ID of object for which to generate downloadable items

    Returns HTTP status 202, accepted.

    Users

    User object

    Attributes

    Parameter Type Description
    name string
    email string
    organization_name string Employer name

    Create user

    Example request

    curl -X POST \
  http://www.rapidskills.org/api/v1/users \
  -H 'Accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
    "data": {
        "type": "user",
        "attributes": {
            "email": "mickey@example.com",
            "name": "Mickey Mouse",
            "organization_name": "Disneyland"
        }
    }
}'

    HTTP Request

    POST http://www.rapidskills.org/api/v1/users

    Query Parameters

    Parameter Type Description
    email
    required    		 string
    name string
    organization_name string Employer name

    Example response

    {
    "data": {
        "id": "3",
        "type": "user",
        "attributes": {
            "email": "mickey@example.com",
            "name": "Mickey Mouse"
        },
        "relationships": {
            "employer": {
                "data": {
                    "id": "2",
                    "type": "organization"
                }
            }
        }
    },
    "included": [
        {
            "id": "2",
            "type": "organization",
            "attributes": {
                "title": "Disneyland"
            }
        }
    ]
}

    Errors

    Errors are returned as an array with the following attributes:

    Parameter Type Description
    status string HTTP status code
    detail string explanation of the problem

    Example request

    curl -X POST \
  http://www.rapidskills.org/api/v1/users \
  -H 'Accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
    "data": {
        "type": "users",
        "attributes": {
            "email": ""
            "name": "Mickey Mouse"
        }
    }
}'

    Example response

    {
    "errors": [
        {
            "status": "422",
            "detail": "Email can't be blank"
        }
    ]
}
