    Introduction

    API documentation for RAPiDSkills. This API follows the JSON:API v1.0 specification. Clients must send all requests with the header Content-Type: application/vnd.api+json without any media type parameters.

    Authentication

    After successful sign in, an access token is returned in the response. All authenticated calls are required to pass this access token as a Bearer token in the header of the request. The tokens do not expire, but signing out or changing the user password will invalidate the token.

    Sign in

    Example request

    curl -X POST \
  http://www.rapidskills.org/api/v1/sessions \
  -H 'Accept: application/vnd.api+json' \
  -H 'Content-Type: application/vnd.api+json' \
  -d '{
    "data": {
        "type": "sessions",
        "attributes": {
            "email": "mickey@example.com",
            "password": "supersecret"
        }
    }
}'

    HTTP Request

    POST http://www.rapidskills.org/api/v1/sessions

    password

    Downloads

    PDF/Excel generation

    This endpoint will trigger the PDF and/or Excel document generation for the resource passed in the relationships field.

    Example request

    curl -X POST \
  http://www.rapidskills.org/api/v1/downloads \
  -H 'Accept: application/vnd.api+json' \
  -H 'Content-Type: application/vnd.api+json' \
  -d '{
    "data": {
        "type": "download",
        "relationships": {
            "downloadable": {
                "data": {
                    "type": "occupation_standard",
                    "id": "1"
                }
            }
        }
    }
}'

    HTTP Request

    POST http://www.rapidskills.org/api/v1/downloads

    Downloadable relationship attributes

    Parameter Type Description
    type
    required    		 string Singular object type for which to generate downloadable items
    id
    required    		 string ID of object for which to generate downloadable items

    Returns HTTP status 202, accepted.

    Parameter Type Description
    status string HTTP status code
